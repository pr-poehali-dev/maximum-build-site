# Список созданных файлов для развертывания

## Файлы конфигурации сервера

### 1. `/public/.htaccess`
**Назначение:** Конфигурация для Apache сервера  
**Будет скопирован в:** `dist/.htaccess` при сборке  
**Функции:**
- URL rewriting для React Router
- GZIP сжатие
- Кэширование статических файлов
- Поддержка HTML5 History API

**Когда использовать:** Если Open Server использует Apache (по умолчанию)

---

### 2. `/public/web.config`
**Назначение:** Конфигурация для IIS сервера  
**Будет скопирован в:** `dist/web.config` при сборке  
**Функции:**
- URL rewriting правила для IIS
- MIME types
- HTTP сжатие
- Поддержка современных форматов файлов

**Когда использовать:** Если Open Server использует IIS

---

## Скрипты автоматизации

### 3. `/build-for-open-server.bat`
**Назначение:** Автоматическая сборка для Windows  
**Язык:** Batch Script  
**Действия:**
1. Установка зависимостей (npm install)
2. Сборка проекта (npm run build)
3. Вывод инструкций по развертыванию

**Как использовать:**
```bash
# Просто дважды кликните по файлу
# Или запустите в командной строке:
build-for-open-server.bat
```

---

### 4. `/build-for-open-server.sh`
**Назначение:** Автоматическая сборка для Unix/Linux/Mac  
**Язык:** Bash Script  
**Действия:** Аналогично .bat версии

**Как использовать:**
```bash
chmod +x build-for-open-server.sh
./build-for-open-server.sh
```

## Как использовать эти файлы

### Сценарий 1: Первое развертывание

1. Прочитайте **OPEN_SERVER_README.md** (главное руководство)
2. Запустите **build-for-open-server.bat** (или .sh)
3. Следуйте инструкциям из **QUICK_START.txt**
4. При проблемах - открывайте **TROUBLESHOOTING.md**

### Сценарий 2: Технический специалист

1. Изучите **BUILD_GUIDE.md** для понимания архитектуры
2. Прочитайте **DEPLOYMENT_SUMMARY.md** для полной картины
3. Настройте сервер согласно рекомендациям
4. Используйте **TROUBLESHOOTING.md** для диагностики

### Сценарий 3: Быстрое развертывание

1. Откройте **QUICK_START.txt**
2. Следуйте 5 шагам
3. Используйте контрольный список
4. Готово!

### Сценарий 4: Возникла проблема

1. Откройте **TROUBLESHOOTING.md**
2. Найдите вашу проблему в оглавлении
3. Следуйте решению step-by-step
4. Проверьте контрольный список диагностики
